Executive Summary

What We Are Building

LienMaster V2 is a fully custom, cloud-native clinical management platform built exclusively for Precision Medical's personal injury (PI) practice model in Utah. It replaces the previous LienMaster legacy system with a modern, HIPAA-aligned, multi-portal architecture — connecting patients, doctors, billing staff, front-office coordinators, attorneys, and administrators in a single integrated ecosystem.

2 apps are live in production today — Super Admin (admin.lienmaster.net) and Time Clock (pmtc.lienmaster.net). Back-Office / Clinic portal (clinic.lienmaster.net) is actively deployed and in final validation. Doctor, Patient, and Attorney portals are in structured development under the same monorepo.

Compliance & Security

HIPAA Compliance Strategy

The platform was architected from day one to be HIPAA-ready. We are currently in Phase 0 (foundation), operating with zero real Protected Health Information (PHI) in the system. PHI is introduced only after all required Business Associate Agreements (BAAs) are in place and a HIPAA consultant review is completed.

🔒
Current status: Zero PHI in production — Phase 0 Foundation
The system is structurally HIPAA-ready: audit logs, role-based access, and encrypted storage are all in place. No real patient data is processed until Phase 1 PHI gate is cleared — which requires signed BAAs with Supabase, Vercel, Sentry, and a HIPAA consultant sign-off.
Safeguard What We Have Status
Audit Logging Every create, update, delete, and delete operation is recorded in an append-only audit log — actor, IP, timestamp, before/after values Done
Role-Based Access Control 7 distinct roles (Super Admin, Admin, Contador, Employee, Provider, Lawyer, AI Auditor) — each sees only what they are permitted to see Done
Data Encryption at Rest Supabase (PostgreSQL on AWS) encrypts all data at rest. SSL/TLS enforced on all connections Done
Data Encryption in Transit All traffic over HTTPS. Let's Encrypt SSL certificates. No plain HTTP allowed on any domain Done
Authentication Supabase Auth — bcrypt hashed passwords, JWT tokens with 1-hour TTL, session invalidation on role change Done
Minimum Necessary Access Each app and each role only has access to the data it needs. Billing staff cannot see clinical notes. Doctors cannot see financial data Done
TV Lobby Display (HIPAA) Waiting room screen shows only anonymized initials + 2 digits — never full patient names. Server-side anonymization, no PHI in the browser Done
BAA with Supabase (database host) Supabase offers a HIPAA BAA for paid plans. Required before processing real PHI Required for PHI
BAA with Vercel (hosting) Vercel Pro/Enterprise plans offer HIPAA BAA. Required before processing real PHI Required for PHI
BAA with Sentry (error monitoring) Sentry offers a HIPAA BAA. Must be signed before any PHI flows through error logs Required for PHI
HIPAA Consultant Review Third-party HIPAA consultant review of the platform before PHI activation. Risk assessment + gap analysis Blocking Gate
SMS Communications (Twilio) Twilio offers HIPAA-compliant messaging with self-serve BAA. Recommended to replace current Weave phone system Under evaluation
Technology

The Stack We Are Building On

Every technology choice was made for long-term stability, type safety, and team velocity. The entire system is 100% TypeScript — from the database schema to the user interface — which eliminates entire categories of runtime bugs.

🏗️

Turborepo

Monorepo — all 6 apps and shared packages in one repository. Shared auth, UI components, and DB layer.

⚛️

Next.js 15

App Router — server-side rendering, edge middleware, and file-based routing for all 6 portals.

🛡️

TypeScript

100% typed codebase — API contracts, DB schemas, and UI all share the same type definitions.

🗄️

Prisma ORM

Type-safe database access layer. Schema-first: DB structure is the single source of truth.

☁️

Supabase

PostgreSQL on AWS us-east-2. Auth, real-time subscriptions, Row Level Security (RLS) policies.

🔗

tRPC

Type-safe API layer — the frontend and backend share the same type contract. Zero API docs needed.

🎨

shadcn/ui + Tailwind

Custom design system (ui-phoenix) built on shadcn primitives. Consistent look across all portals.

🚀

Vercel

Each app deploys independently to Vercel. Auto-deploy on git push to main. Global CDN edge network.

🔁

GitHub CI/CD

GitHub Actions pipeline — lint, typecheck, and build validation on every pull request before deploy.

🐛

Sentry

Error monitoring and performance tracking for production. Real-time alerts on application errors.

📩

Resend

Transactional email — password resets, salary alerts, and system notifications sent via Resend API.

📱

PWA Support

Super Admin and Time Clock are installable as apps on any device — phone, tablet, or desktop.
