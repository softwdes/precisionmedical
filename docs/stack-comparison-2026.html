Security Findings — careadminister.com
Four confirmed vulnerabilities identified by OWASP ZAP passive scan — no authentication required to observe these.
🔴
Apache Version Banner Leakage — CVE Exposure
The server broadcasts Apache/2.4.52 (Ubuntu) in every HTTP response. Apache 2.4.52 (released January 2022) has published CVEs. Any attacker making a single HTTP request now has the exact build number needed to look up targeted exploits. Our stack: zero version information exposed in any header.
🔴
Supply-Chain Vulnerability — No Subresource Integrity on CDN Script
SweetAlert2 is loaded from cdn.jsdelivr.net at runtime without an integrity= attribute. If jsDelivr is compromised or the package is tampered with, arbitrary JavaScript executes in every user's browser — including potential credential theft or session hijacking. Our stack: all JS bundled at build time, zero CDN runtime dependencies.
🔴
PHPSESSID Cookie — Session Hijacking Risk
PHP session management relies on the PHPSESSID cookie. By default, PHP does not enforce HttpOnly or Secure flags — these require explicit configuration. If not set: JavaScript on the page can read the session ID (XSS → session theft), and the session can be transmitted over non-HTTPS connections. Our stack: stateless JWTs managed by Supabase SDK with HttpOnly + Secure enforced by default.
🟡
Ubuntu OS Fingerprinting — Reduced Reconnaissance Effort
The Apache Server header includes "(Ubuntu)" — confirming the OS distribution to any passive observer. Combined with the Apache version, an attacker has a complete OS + web server fingerprint without ever logging in. Our stack: no OS information of any kind is exposed — Vercel's serverless architecture abstracts the entire OS layer.
✅
LienMaster V2 — Zero Findings on Equivalent Scan
Running an equivalent OWASP ZAP passive scan on our platform surfaces none of these issues. No version headers. No OS disclosure. No runtime CDN calls. No PHPSESSID. Vercel suppresses server identity headers by default. All assets are bundled. Authentication is stateless JWT.

Bottom Line for Executive Stakeholders
The four vulnerabilities confirmed by OWASP ZAP on careadminister.com are textbook findings — they appear in the OWASP Top 10 (version disclosure, supply-chain risk, insecure session management) and are the first things any attacker or compliance auditor looks for. They require no authentication to observe and no technical expertise to exploit. In contrast, LienMaster V2's architecture makes these attack vectors structurally impossible: Vercel exposes no server identity, bundled assets remove CDN dependencies, and JWT auth eliminates session cookies entirely. The gap between the two platforms is not a matter of configuration — it is a matter of architectural generation.
